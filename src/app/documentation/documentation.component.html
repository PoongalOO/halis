Mermaid permet de générer divers diagrammes à partir d'une syntaxe textuelle simple. Voici quelques exemples :

1. Flowchart (Diagramme en flux)

                  graph TD;
                    A[Début] --> B{{ '{' }}Condition ?{{ '}' }};
                    B -- Oui --> C[Action 1];
                    B -- Non --> D[Action 2];
                    C --> E[Fin];
                    D --> E[Fin];

Ce diagramme représente un processus décisionnel avec une condition.

2. Sequence Diagram (Diagramme de séquence)

                  sequenceDiagram
                    participant Alice
                    participant Bob
                    Alice->>Bob: Salut Bob, comment vas-tu ?
                    Bob-->>Alice: Très bien, merci !

Ce diagramme montre l'échange de messages entre deux participants.

3. Gantt Diagram (Diagramme de Gantt)

                  gantt
                    title Projet Exemple
                    dateFormat  YYYY-MM-DD
                    section Développement
                    Planification      :a1, 2024-04-01, 10d
                    Développement      :after a1, 20d
                    Tests              :2024-05-01, 5d

Ce diagramme présente une planification de projet avec des durées et des dépendances.

4. Class Diagram (Diagramme de classes)

                  classDiagram
                    Animal <|-- Chat
                    Animal <|-- Chien
                    Animal : +String nom
                    Animal : +int age
                    Animal : +void manger()
                    Animal : +void dormir()

Ce diagramme illustre une hiérarchie de classes.

Pour utiliser Mermaid, il suffit de copier le code correspondant dans un fichier ou un composant qui se charge de le rendre (comme dans l'exemple d'intégration précédemment présenté).

Marp permet de créer des présentations à partir de fichiers Markdown en convertissant le contenu en diapositives. Voici quelques exemples :

1. Présentation simple

 --- marp: true title: "Présentation Exemple" ---
                  Slide 1
                  
                  Contenu de la première diapositive
                  Slide 2
                  
                  Contenu de la deuxième diapositive

Cette présentation basique montre comment structurer un document Markdown pour générer une série de diapositives.

2. Présentation avec thèmes

 --- marp: true theme: gaia title: "Présentation Thématique" ---
                  Titre de la présentation
                  Sous-titre
                  Diapositive Thématique
                  
                  Utilisez des thèmes pour personnaliser l'apparence de vos diapositives.

Cet exemple illustre l'utilisation de thèmes pour modifier le design de la présentation.

Markmap permet de générer des cartes heuristiques à partir d'un document Markdown. Voici quelques exemples :

1. Carte heuristique de base

 # Carte Mentale Exemple ## Sujet Principal - Idée 1 - Sous-idée 1.1 - Sous-idée 1.2 - Idée 2

Ce schéma montre comment organiser visuellement des idées en hiérarchie grâce à Markdown.

2. Carte détaillée d'un projet

 # Projet ## Phase 1: Planification - Objectif 1 - Objectif 2 ## Phase 2: Exécution - Tâche 1 - Détail A - Détail B - Tâche 2

Ce diagramme illustre la structuration d'un projet en différentes phases et tâches.

easyMD est un outil convivial pour éditer et visualiser du Markdown. Voici quelques exemples :

1. Document Markdown simple

 # Titre du Document Ceci est un exemple de document Markdown édité avec easyMD.
                  Section
                  
                      Élément 1
                  
                      Élément 2

Ce document présente les bases de la rédaction en Markdown avec des titres et des listes.

2. Document avec code et tableau

                  
                  Exemple de Code
                  
                  Voici un extrait de code en Python :
                  
                  def bonjour():
                      print("Bonjour, monde !")
                  
                  Tableau
                  Nom	Âge
                  Alice	30
                  Bob	25

Ce document montre comment intégrer du code source et des tableaux dans un fichier Markdown.

Pour utiliser Marp, Markmap et easyMD, il suffit de copier le code correspondant dans un fichier ou un composant dédié à leur rendu.
